# Dataset Description (Theoretical + Empirical)

## 1. Purpose of This Dataset
This dataset represents personal financial transaction behavior over time and is suitable for:
- transaction category classification,
- subtype classification,
- spending pattern analysis,
- daily and monthly forecasting support.

From a machine learning perspective, each transaction can be viewed as a behavioral event generated by a latent process that depends on intent (payment/transfer), context (merchant/counterparty), and temporal factors (day, month, pay-cycle, recharge cycles).

## 3. Column-Level Description (Conceptual)

### `date` (temporal feature)
Represents event time and should theoretically capture periodicity:
- daily cycles (food/commute micro-spending),
- weekly cycles (weekend intensity),
- monthly cycles (salary + recurring bills + recharge patterns).

### `description` (text signal)
Primary semantic signal for category modeling. Theoretically, descriptions encode:
- merchant/counterparty identity,
- transfer channel tokens,
- contextual keywords.

In practice, this behaves like sparse text with strong local keywords and high lexical variation.

### `amount` (continuous signal)
Heavy-tailed monetary variable with outliers. The mean-median gap indicates:
- many low-value routine transactions,
- fewer high-value sporadic events.

Theoretical implication: this resembles a right-skewed mixture distribution (routine + occasional large payments).

### `type` and `subtype` (structural labels)
These encode transactional mechanics (debit/credit, transfer directions, withdrawals). They are useful as auxiliary targets or conditioning features in hierarchical models.

### `category`, `category_source`, `confidence` (labeling metadata)
These represent current labeling state and quality source. Since current labels are predominantly keyword-derived, theoretical label noise is tied to rule coverage and lexical ambiguity.

## 4. Theoretical Data Behavior and Trends

## 4.1 Class imbalance trend
The category process is strongly long-tailed with dominant `other` class. Theoretically, a classifier trained naively will optimize global accuracy by over-predicting `other`, reducing minority recall.

## 4.2 Amount trend
Given median (25) << mean (478), transactions likely follow a right-skewed distribution with bursty high-value events. Models using raw amounts may overfit large-value anomalies unless transformed.

## 4.3 Temporal trend
With 264 active days and ~5.7 events/day on average, event density is moderate but potentially uneven by weekday/month. Forecasting should expect non-stationary segments around salary windows, bill dates, or seasonality.

## 4.4 Text-to-label trend
Because labels are currently keyword-driven, learning is expected to capture lexical anchors first (high precision on known merchants) and struggle on unseen/ambiguous descriptions unless robust text features are used.

## 4.5 Source generalization trend
Data originates from a single source file. Theoretically, domain shift risk is high when new statement formats, new counterparties, or OCR/token changes appear.

---

## 5. Statistical Character (Modeling View)
A practical theoretical framing:

- Let $X_t = (\text{text}, \text{amount}, \text{time features}, \text{type/subtype})$.
- Let $Y_t = \text{category}$.
- The conditional process $P(Y_t \mid X_t)$ is expected to be:
  - highly imbalanced,
  - partly deterministic for known keywords,
  - uncertain for sparse/unseen descriptions,
  - weakly time-conditioned for category but strongly time-conditioned for volume/amount forecasting.

For forecasting tasks, daily spend can be treated as a non-negative time series with intermittent spikes and calendar effects.

## 6. Strengths of Current Dataset
- Clean core schema with no missing values in key transaction fields.
- Sufficient transaction count for baseline text classification experiments.
- Daily and monthly aggregates already available for forecasting baselines.
- Explicit subtype/type metadata supports multi-task or hierarchical learning.

---

## 7. Constraints and Theoretical Risks
- Severe category imbalance (`other` dominance).
- Limited label diversity (5 categories currently observed).
- Single-source domain (weak out-of-distribution robustness).
- Label generation mostly rule-based (`category_source=keyword`), potentially propagating rule bias into learned models.
- Heavy-tailed amount distribution can destabilize regression losses without transformation.

---

## 8. Recommended Feature Engineering (Theory-Aligned)

### Text features
- Character n-grams and word n-grams (robust to formatting noise).
- Merchant/counterparty token extraction.
- Normalization: uppercase, punctuation cleanup, repeated-space reduction.

### Numeric features
- `log1p(amount)` for stability.
- Quantile bins for robust class boundary cues.

### Time features
- day-of-week, day-of-month, month, is-month-start/end.
- optional lag/rolling features for forecasting.

### Label strategy
- treat `other` as fallback class with calibrated thresholding,
- use class weighting or focal loss for minority categories,
- evaluate macro-F1 in addition to accuracy.

---

## 9. Model Planning Guidance

### For category classification
Use a text-first classifier with calibrated probabilities and imbalance-aware training. Start with baseline linear models (TF-IDF + Logistic Regression), then evaluate compact transformer variants if needed.

### For subtype classification
Subtype appears highly deterministic from description patterns; a lightweight rule+model hybrid can be effective.

### For spending forecast
Model daily totals as a skewed non-negative process. Gradient boosting or regularized regression with calendar features is a practical baseline.

---

## 10. Suggested Evaluation Protocol
- Time-aware split (train on earlier period, validate on later period).
- Report:
  - Macro-F1 and per-class recall for classification,
  - MAE/RMSE/MAPE (careful with near-zero days) for forecasting.
- Monitor minority classes (`breakfast`, `recharge`, `lunch`) explicitly.

---

## 11. Summary
The dataset is clean and structured, with meaningful behavioral signals in text, amount, and time. The dominant theoretical challenge is imbalance plus single-source generalization. With imbalance-aware learning, robust text features, and time-aware validation, the dataset is suitable for baseline-to-intermediate transaction intelligence models.
